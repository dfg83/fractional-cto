Review the current code, component, screenshot, or visual artifact against the visual-design-principles skills.

Follow this process:

1. Identify which visual design areas are relevant to the current context (layout, typography, color, whitespace, hierarchy, consistency, craftsmanship, expression, responsive, accessibility)
2. For each relevant area, invoke the corresponding visual-design-principles skill
3. Evaluate the current artifact against each skill's review checklist
4. If reviewing code (HTML/CSS/JSX), check for specific measurable violations (contrast ratios, spacing values, font sizes, image optimization)
5. If reviewing a screenshot or visual description, evaluate against the observable indicators and scoring rubrics
6. Report findings organized by principle, using this format for each:

**[Principle Name]** (Score: X/5)
- Violations found (with specific file/line references for code, or visual location for screenshots)
- What to fix and how
- Items that already comply

7. Provide a summary with:
   - 8-dimension score breakdown (Layout, Typography, Color, Whitespace, Hierarchy, Consistency, Craftsmanship, Expression) — each scored 1-5
   - Total score out of 40
   - Total violations by severity (critical / important / suggestion)
   - Top 3 highest-impact improvements to make first

Focus on actionable, specific feedback. Reference the exact principle and research being violated.
